# AppBar布局调整说明

## 🎯 需求理解
用户希望将历史记录按钮：
- 从右上角移到左上角
- 图标从历史记录图标改为三横杠菜单图标
- 位置放在"马上码"标题的左侧

## ✅ 实现方案

### 📍 **布局变化**
```dart
// 修改前 (右上角)
AppBar(
  title: Text('马上码'),
  actions: [
    IconButton(Icons.refresh),      // 刷新剪贴板
    IconButton(Icons.history),      // 历史记录 ❌
  ],
)

// 修改后 (左上角)
AppBar(
  leading: IconButton(Icons.menu),       // 历史记录 ✅
  title: Text('马上码'),              // 标题居中
  actions: [
    IconButton(Icons.refresh),      // 刷新剪贴板
  ],
)
```

### 🔧 **具体改动**

#### 1. **位置调整**
- **之前**: `actions` 列表 (右上角)
- **修改后**: `leading` 属性 (左上角)

#### 2. **图标更换**
- **之前**: `Icons.history` (历史记录图标)
- **修改后**: `Icons.menu` (三横杠菜单图标)

#### 3. **布局优化**
- **leading**: 历史记录按钮 (左上角)
- **title**: "马上码" (居中显示)
- **actions**: 刷新剪贴板按钮 (右上角)

## 🎨 视觉效果

### 📱 新的界面布局
```
[三横杠]           马上码           [刷新]
   ↑                 ↑              ↑
历史记录            应用标题        刷新剪贴板
```

### 🎯 用户体验提升
- **符合习惯**: 左上角菜单符合主流应用设计
- **操作便利**: 左手操作更容易触及历史记录
- **视觉平衡**: 左右各一按钮，布局更均衡
- **功能明确**: 菜单图标更直观表达"更多功能"

## 🔍 设计考虑

### 1. **图标选择**
```dart
Icons.menu  // 三横杠菜单图标
- 通用性：用户熟悉的菜单符号
- 直观性：表示"更多选项"或"历史"
- 简洁性：设计简洁，识别度高
```

### 2. **位置逻辑**
```dart
leading: IconButton(...)
- 左上角通常是导航或菜单位置
- 符合Material Design规范
- 与主流应用保持一致
```

### 3. **功能映射**
```dart
// 三横杠菜单 → 历史记录功能
- 既视为主导航菜单
- 又是历史记录入口
- 一键多用，操作高效
```
